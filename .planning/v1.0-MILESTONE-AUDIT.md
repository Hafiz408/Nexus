### Integration Findings

#### Finding 1: Incremental re-index FTS5 stale row issue (PIPE-03, STORE-03, EMBED-05)

- **Severity:** Non-critical tech debt
- **Path:** `pipeline.py:57` → `delete_nodes_for_files()` → cleans `graph_nodes` + `graph_edges` only
- **Gap:** No per-file deletion from `code_fts` (FTS5) or `code_embeddings` (pgvector) for changed files
- **Impact:** If a function is renamed, the old FTS5 row persists (stale exact-match result). pgvector is safe via ON CONFLICT upsert. Affects only incremental re-index of renamed/removed functions.
- **Fix:** Add a `delete_embeddings_for_files(file_paths, repo_path)` function to `embedder.py` and call it in the incremental branch of `run_ingestion()` alongside `delete_nodes_for_files()`.

#### Finding 2: Backend service has no Docker healthcheck (INFRA-01)

- **Severity:** Non-critical tech debt
- **Path:** `docker-compose.yml` backend service definition
- **Gap:** `postgres` has `pg_isready` healthcheck; `backend` has none. `GET /health` exists but Docker cannot expose backend health status.
- **Impact:** External orchestrators (CI, future VS Code extension activation gate) cannot rely on `docker compose ps` backend health state.

#### Finding 3: load_graph() has no production consumer (STORE-02)

- **Severity:** Informational (by design)
- **Path:** `graph_store.py:125` — fully implemented but uncalled in phases 1-7
- **Context:** Phase 8 (Graph RAG) will consume `load_graph()` for retrieval. Deferred integration is intentional.
