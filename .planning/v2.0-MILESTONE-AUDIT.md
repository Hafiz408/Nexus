## Integration Gaps

### INT-01 (Critical): Extension never sends target_node_id

The Review and Test E2E flows are broken. The extension POSTs `{query, repo_path, intent_hint}` only.

**Root cause:** Four fields are missing from the POST body in `SseStream.ts` and not declared in `types.ts`:
- `target_node_id` — required by reviewer and tester to locate the subject node in the graph
- `selected_file` / `selected_range` — required by reviewer for range-targeted review (REVW-03)
- `repo_root` — required to resolve absolute paths

**Crash points:**
```
reviewer.py:89  → _assemble_context(G, None)   → ValueError: target_node_id required
tester.py:117   → _get_callees(G, None)         → ValueError: target_node_id required
```

**Files to fix:**
| File | Change needed |
|---|---|
| `extension/src/types.ts` | Add `target_node_id?`, `selected_file?`, `selected_range?`, `repo_root?` to query message type |
| `extension/src/SseStream.ts` | Include fields in POST body when provided |
| `extension/src/SidebarProvider.ts` | Forward fields from webview message to streamQuery |
| `backend/app/agent/reviewer.py` | Guard: raise informative error if target_node_id absent vs. crash |
| `backend/app/agent/tester.py` | Guard: raise informative error if target_node_id absent vs. crash |

**Affected requirements:** REVW-01, REVW-02, REVW-03, TEST-01, TEST-02, TEST-03, TEST-04, TEST-05, MCP-01, MCP-03, EXT-06, EXT-07, EXT-08, EXT-09

---

### INT-02 (Major): MCP tools have no production call site

Both `post_review_comments` and `write_test_file` are fully implemented (all 6 MCP requirements pass in unit tests). The extension `SidebarProvider.ts` contains a TODO stub for the GitHub PR post action, explicitly deferred to Phase 27. `write_test_file` is unreachable because the tester crashes before returning output.

This gap is **downstream of INT-01** — fixing INT-01 unblocks the MCP layer.

## Gap Closure Scope

A single phase can close all critical gaps:

1. **Extension** — add `target_node_id`, `selected_file`, `selected_range`, `repo_root` to the query message type and POST body. Add UI to let the user select the target node (command palette, editor selection, or inline input).
2. **Backend guards** — improve error handling in `reviewer.py` and `tester.py` so missing `target_node_id` returns a helpful error message instead of crashing.
3. **MCP call site** — wire `post_review_comments` invoke in `SidebarProvider.ts` when Review response arrives.

This unlocks all 13 partial requirements and both broken E2E flows.
