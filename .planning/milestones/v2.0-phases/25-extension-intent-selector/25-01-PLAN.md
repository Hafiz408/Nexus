```typescript
export async function streamQuery(
  question: string,
  repoPath: string,
  webview: vscode.Webview,
  backendUrl: string,
  onCitations?: (citations: Citation[]) => void,
  intentHint?: string,

```

In the `body: JSON.stringify({...})` call, add a conditional spread after `hop_depth`:
```typescript
body: JSON.stringify({
```

CRITICAL: Use `intentHint ? { intent_hint: intentHint } : {}` — the spread is only added when intentHint is a non-empty truthy string. Never send `intent_hint: "auto"`.

```typescript
  debug:   'Debug',
```

**2. Add selectedIntent state** — Inside the `App()` function, after the existing `const [expandedCitations, ...]` line, add:

```typescript
const [selectedIntent, setSelectedIntent] = useState<IntentOption>('auto');
```

**3. Update handleSend** — The current postMessage call is:
```typescript

```
Replace with:
```typescript

  type: 'query',

```
CRITICAL: When `selectedIntent === 'auto'`, the field must be `undefined` (omitted from JSON.stringify output), NOT the string `"auto"`. The backend V2 gate is `intent_hint not None and not "auto"` — sending `"auto"` silently degrades to V1 path.

Also add `selectedIntent` to the `useCallback` dependency array of `handleSend`:
```typescript
}, [inputValue, isStreaming, addLog, selectedIntent]);
```

**4. Add intent selector JSX** — In the render return, inside `.chat-body`, insert a `<div className="intent-selector">` block immediately BEFORE the `<div className="input-area">` element (as a sibling, not nested inside it):

```tsx
<div className="intent-selector">
  {INTENT_OPTIONS.map((intent) => (
    <button
      key={intent}
      className={`intent-pill${selectedIntent === intent ? ' active' : ''}`}
      onClick={() => setSelectedIntent(intent)}
      disabled={isStreaming}
      title={INTENT_LABELS[intent]}
    >
      {INTENT_LABELS[intent]}
    </button>
  ))}
</div>
```

Note: Use `INTENT_LABELS[intent]` for the button text — this already returns "Ask" for auto, so no special-casing needed.
