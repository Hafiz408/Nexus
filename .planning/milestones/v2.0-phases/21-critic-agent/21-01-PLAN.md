**CriticResult model**:
```python
class CriticResult(BaseModel):
    """Quality gate result from a single critique() call."""
    score: float = Field(ge=0.0, le=1.0)          # 0.4*G + 0.35*R + 0.25*A
    groundedness: float = Field(ge=0.0, le=1.0)   # fraction of cited nodes in retrieved set
    relevance: float = Field(ge=0.0, le=1.0)      # structural content quality
    actionability: float = Field(ge=0.0, le=1.0)  # specificity (file:line, suggestions)
    passed: bool                                    # True = accept; False = route back
    feedback: str | None                           # None when passed=True; critique text otherwise
    loop_count: int                                # loop_count value at time of critique
```

**Private helper: `_compute_groundedness(cited_nodes: set[str], retrieved_nodes: list[str]) -> float`**
- Returns `1.0` when `cited_nodes` is empty (nothing cited = fully grounded; avoids div-by-zero)
- Otherwise: `len(cited_nodes & set(retrieved_nodes)) / len(cited_nodes)`

**Private helper: `_extract_groundedness_inputs(result) -> tuple[set[str], list[str]]`**
- Uses `isinstance` to dispatch on result type — import types lazily inside this function:
  ```python
  from app.agent.debugger import DebugResult   # noqa: PLC0415
  from app.agent.reviewer import ReviewResult  # noqa: PLC0415
  from app.agent.tester import TestResult      # noqa: PLC0415
  ```
- `DebugResult`: cited = `{s.node_id for s in result.suspects}`, retrieved = `result.traversal_path`
- `ReviewResult`: cited = `{f.file_path for f in result.findings}`, retrieved = `result.retrieved_nodes`
- `TestResult`: cited = `set()`, retrieved = `[]`  → groundedness will be 1.0 per empty-set rule
- Unknown type: cited = `set()`, retrieved = `[]`

**Private helper: `_compute_relevance(result) -> float`**
- Dispatch on result type (same lazy imports inside function):
- `DebugResult`: `1.0` if `len(result.suspects) > 0 and result.diagnosis` else `0.3`
- `ReviewResult`: `1.0` if `len(result.findings) > 0 and result.summary` else `0.3`
- `TestResult`: `1.0` if `result.test_code and "def test_" in result.test_code` else `0.3`
- Unknown type: `0.5`

**Private helper: `_compute_actionability(result) -> float`**
- `DebugResult`: `min(len(result.suspects) / 5.0, 1.0)` (actionable if has specific suspects)
- `ReviewResult`: `sum(1 for f in result.findings if f.suggestion) / max(len(result.findings), 1)` — returns `1.0` if findings is empty (nothing to critique = not actionability problem)
- `TestResult`: `min(result.test_code.count("def test_") / 3.0, 1.0)` if test_code else `0.0`
- Unknown type: `0.5`

**Private helper: `_weighted_score(g: float, r: float, a: float) -> float`**
```python
score = WEIGHT_GROUNDEDNESS * g + WEIGHT_RELEVANCE * r + WEIGHT_ACTIONABILITY * a
return round(min(max(score, 0.0), 1.0), 4)
```

**Private helper: `_generate_feedback(g: float, r: float, a: float, score: float) -> str`**
- Build list of `parts` for any sub-score below 0.5, appending specific guidance:
  - g < 0.5: `f"Groundedness is low ({g:.2f}): cited nodes not found in retrieved context. Reference only functions/files that were part of the graph traversal."`
  - r < 0.5: `f"Relevance is low ({r:.2f}): response lacks substantive findings. Ensure the output directly addresses the query with specific findings."`
  - a < 0.5: `f"Actionability is low ({a:.2f}): findings lack concrete details. Include file paths, line numbers, and specific suggestions."`
  - If `parts` is empty (all sub-scores >= 0.5 but composite still below threshold): append `f"Overall score ({score:.2f}) is below threshold. Improve specificity and coverage of findings."`
- Return `" | ".join(parts)`

**Public API: `critique(result, loop_count: int, settings=None) -> CriticResult`**
Full docstring: "Apply deterministic quality gate to a specialist result. loop_count=0 on first call; cap fires when loop_count >= max_critic_loops."

Algorithm:
1. Lazy-load settings if None: `from app.config import get_settings; settings = get_settings()`
2. Read `max_loops = settings.max_critic_loops` and `threshold = settings.critic_threshold`
3. Compute `g`, `r`, `a` using the three helpers above
4. Compute `score = _weighted_score(g, r, a)`
5. **Hard cap check first (CRIT-03):** if `loop_count >= max_loops` → return `CriticResult(score=score, groundedness=g, relevance=r, actionability=a, passed=True, feedback=None, loop_count=loop_count)`
6. **Quality gate (CRIT-02):** if `score < threshold` → `feedback = _generate_feedback(g, r, a, score)` → return `CriticResult(..., passed=False, feedback=feedback, ...)`
7. **Pass:** return `CriticResult(..., passed=True, feedback=None, ...)`
