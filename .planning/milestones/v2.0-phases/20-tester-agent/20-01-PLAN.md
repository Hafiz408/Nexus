System prompt — use UPPERCASE emphasis and mandate ≥3 test functions (pitfall 3 prevention):
```python
TESTER_SYSTEM = """You are a test code generation assistant. \
Given a target function and its callees (dependencies), generate runnable test code.

Framework: {framework}
Mock syntax for this framework:
{mock_syntax}

REQUIREMENTS:
1. Generate EXACTLY three or more test functions: one for the happy path, \
one for error/exception cases, and one for edge cases (empty input, boundary values, etc.).
2. Every callee listed in "Mock targets" must appear as a mock/patch in the test setup.
3. Use correct {framework} syntax — import statements, test runner decorators, assertions.
4. Return ONLY valid, runnable code. No markdown fences. No explanatory text.

Mock targets (callees of the target function):
{mock_targets}"""

TESTER_PROMPT = ChatPromptTemplate.from_messages([
    ("system", TESTER_SYSTEM),
    ("human", "Generate tests for: {question}\n\nTarget function: {target_name}\n"
              "File: {target_file}\nLines: {line_start}-{line_end}"),
])
```

**Private helpers:**

_detect_framework(repo_root: str) -> str:
- root = Path(repo_root)
- For each framework in _FRAMEWORK_MARKERS: check if any marker exists via (root / marker).exists()
- First match wins (priority order: pytest, jest, vitest, junit)
- Fallback: if any(root.rglob("test_*.py")) return "pytest"
- Final fallback: return "unknown"

_get_callees(G: nx.DiGraph, target_id: str) -> list[dict]:
- If target_id not in G: raise ValueError(f"target_node_id {target_id!r} not found in graph")
- Return list of dicts with keys "name" and "file_path" for all CALLS-edge successors
- Use G.nodes[succ].get("name", succ) and G.nodes[succ].get("file_path", "") for safety
- Filter by G.edges[target_id, succ].get("type") == "CALLS"

_derive_test_path(func_name: str, framework: str) -> str:
- name = func_name.replace(".", "_").replace("::", "_")
- pytest → f"tests/test_{name}.py"
- jest → f"__tests__/{name}.test.ts"
- vitest → f"{name}.test.ts"
- junit → f"src/test/java/{name.capitalize()}Test.java"
- unknown/default → f"tests/test_{name}.py"

Step 3 — Callee enumeration (mock targets):

Step 4 — Target node attributes:
```python
if target_node_id not in G:
    raise ValueError(f"target_node_id {target_node_id!r} not found in graph")
attrs = G.nodes[target_node_id]
target_name = attrs.get("name", target_node_id)
target_file = attrs.get("file_path", "")
line_start = attrs.get("line_start", 0)
line_end = attrs.get("line_end", 0)
```

Step 5 — LLM call with structured output (lazy import — CRITICAL: inside function body):
```python
from app.core.model_factory import get_llm  # noqa: PLC0415
llm = get_llm()
structured_llm = llm.with_structured_output(_LLMTestOutput)
prompt = TESTER_PROMPT.partial(
    framework=framework,
    mock_syntax=_MOCK_SYNTAX.get(framework, _MOCK_SYNTAX["unknown"]),
    mock_targets=mock_targets,
)
chain = prompt | structured_llm
llm_output: _LLMTestOutput = chain.invoke({
    "question": question,
    "target_name": target_name,
    "target_file": target_file,
    "line_start": line_start,
    "line_end": line_end,
})
```

Step 6 — Assemble TestResult deterministically (never trust LLM for path or framework):
```python
test_file_path = _derive_test_path(target_name, framework)
return TestResult(
    test_code=llm_output.test_code,
    test_file_path=test_file_path,
    framework=framework,
)
```
