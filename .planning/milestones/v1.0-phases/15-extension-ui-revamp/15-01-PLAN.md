<task type="auto">
  <name>Task 1: Textarea auto-grow</name>
  <files>extension/src/webview/App.tsx, extension/src/webview/index.css</files>
  <action>
In App.tsx:

1. Add `textareaRef` ref alongside `messagesEndRef`:
   ```typescript
   const textareaRef = useRef<HTMLTextAreaElement>(null);
   ```

2. Add `handleInputChange` callback replacing the inline `onChange` on textarea. The handler MUST read scrollHeight from `e.target` synchronously BEFORE React state update — do NOT read from ref after setInputValue:
   ```typescript
   const handleInputChange = useCallback(
     (e: React.ChangeEvent<HTMLTextAreaElement>): void => {
       setInputValue(e.target.value);
       const el = e.target;
       el.style.height = 'auto';
       el.style.height = `${el.scrollHeight}px`;
     },
     []
   );
   ```

3. In `handleSend`, after `setInputValue('')`, reset textarea height:
   ```typescript
   if (textareaRef.current) {
     textareaRef.current.style.height = 'auto';
   }
   ```

4. Update the `<textarea>` JSX element:
   - Add `ref={textareaRef}`
   - Remove `rows={1}` attribute
   - Change `onChange` from `(e) => setInputValue(e.target.value)` to `onChange={handleInputChange}`

In index.css, update `.input-area textarea`:
   - Remove `min-height: 32px` (it fights auto-grow: when height resets to 'auto', min-height prevents the reset from working, causing an additive growth loop)
   - Keep `max-height: 100px` and `overflow-y: auto` (these stay)
   - Add `overflow-y: hidden` initially so scrollbar only appears at max-height — change to `overflow-y: auto` when height hits max-height. Actually: keep `overflow-y: auto` as-is since CSS handles this correctly with max-height.

Anti-patterns to avoid:
- Do NOT use ResizeObserver (async feedback loop)
- Do NOT use useEffect([inputValue]) to drive height (deferred, causes flicker)
- Do NOT read scrollHeight from ref after setInputValue (stale DOM)
- Do NOT remove `max-height: 100px` from CSS (this is the 5-row cap)
- Verify `box-sizing: border-box` is already on the global `*` selector — it is, so no additive loop
