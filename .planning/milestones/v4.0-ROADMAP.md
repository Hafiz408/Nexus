# v4.0 Model & Provider Agnostic + Zero-Setup Distribution — Archive (Shipped: 2026-04-01)

## Milestone Goal
Make Nexus usable by anyone — users bring their own AI provider, configure it in VS Code settings, and the extension works out-of-the-box with a bundled backend sidecar (no manual Python setup).

## Phases (30–35)
- Phase 30: backend-config — Dynamic in-memory config endpoint + health endpoint + 5-provider model factory (completed 2026-04-01)
- Phase 31: embedding-safety — nexus_meta table tracks embedding provider/model; mismatch detection on config push (completed 2026-04-01)
- Phase 32: extension-settings-keys — VS Code settings for provider/model + SecretStorage API key commands + config push (completed 2026-04-01)
- Phase 33: sidecar-manager — SidecarManager spawns/kills bundled backend binary; output channel; dev-mode skip (completed 2026-04-01)
- Phase 34: reindex-guard-status — Reindex guard blocks chat until valid index; webview status bar shows active config (completed 2026-04-01)
- Phase 35: build-pipeline — PyInstaller Mac + Windows binaries; GitHub Actions CI builds and packages into .vsix (completed 2026-04-01)

## Key Accomplishments
- POST /api/config dynamic in-memory config replaces .env singleton
- 5-provider model factory: OpenAI, Mistral, Anthropic (chat only), Ollama (local), Gemini
- SidecarManager: spawns/kills binary, polls /health before config push, lockfile-based multi-window reuse
- SecretStorage API key management — keys never written to disk
- nexus_meta mismatch detection + chat guard until valid index exists
- GitHub Actions pipeline: 2 parallel PyInstaller jobs (Mac + Windows) → .vsix
