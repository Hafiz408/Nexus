The new implementation follows this flow:

1. **Platform guard (unchanged):**
   ```
   const archiveName = this._archiveName();
   const executableName = this._executableName();
   if (!archiveName || !executableName) { return undefined; }
   ```

   ```
   const cacheDir = path.join(path.dirname(this._lockfilePath), 'backend', version);
   const executablePath = path.join(cacheDir, executableName);
   if (fs.existsSync(executablePath)) {
     this._channel.appendLine(`[SidecarManager] Using cached backend at ${executablePath}`);
     return executablePath;

   ```

3. **Download URL construction (per CONTEXT.md locked decision):**

   ```
   const baseUrl = `https://github.com/Hafiz408/Nexus/releases/download/v${version}`;
   const archiveUrl = `${baseUrl}/${archiveName}`;
   ```

4. **Fetch checksum (sequential, before download):**
   ```
   let expectedHash: string;
   try {
     expectedHash = await this._fetchChecksum(baseUrl, archiveName);
   } catch (err) {
     // Dev build / no matching release: 404 is non-fatal per CONTEXT.md
     this._channel.appendLine(`[SidecarManager] Checksum fetch failed: ${err instanceof Error ? err.message : String(err)}. No GitHub Release for v${version}?`);
   ```

5. **Download with progress notification (per CONTEXT.md locked decision: "Downloading Nexus backend..."):**

   ```

       cancellable: false,

       await this._downloadAndVerify(archiveUrl, tmpPath, expectedHash, progress);
   ```

   ```
   this._channel.appendLine(`[SidecarManager] Extracting backend archive to ${cacheDir} ...`);
   await new Promise<void>((resolve, reject) => {
     const tarArgs = ['-xzf', tmpPath, '-C', cacheDir];
     const tar = cp.execFile('tar', tarArgs, { stdio: 'pipe' });
     tar.on('close', (code) => {
       if (code === 0) { resolve(); }
       else { reject(new Error(`tar exited with code ${code}`)); }
     });
     tar.on('error', reject);
   });
   ```


8. **Return executable path:**
   ```
   return executablePath;
   ```

  <action>
Modify the `start()` method to catch errors thrown by `_ensureExtracted` during the download flow and surface them to the user via `_showDownloadError`.

Currently in `start()`, the call to `_ensureExtracted` (around line 198) looks like:
```
const binaryPath = await this._ensureExtracted(version);
if (!binaryPath) {
  this._backendUrl = 'http://127.0.0.1:8000';
  return this._backendUrl;
}
```

Replace this block with a try/catch:
```
let binaryPath: string | undefined;
try {
  binaryPath = await this._ensureExtracted(version);
} catch (err) {
  // Download or checksum failure on cold path — show error notification per CONTEXT.md
  const errMsg = err instanceof Error ? err.message : String(err);
  this._channel.appendLine(`[SidecarManager] Download failed: ${errMsg}`);
  await this._showDownloadError(errMsg);
  this._backendUrl = 'http://127.0.0.1:8000';
  return this._backendUrl;
}
if (!binaryPath) {
  // Unsupported platform or dev-build 404 — silent fallback to port 8000
  this._backendUrl = 'http://127.0.0.1:8000';
  return this._backendUrl;
}
```

This satisfies DIST-04: download failure shows error notification with GitHub Releases link.

The `if (!binaryPath)` fallback handles two cases:
- Unsupported platform (archiveName is undefined) — silent, same as before (PRES-03 compatible)
- Dev-build 404 (checksum fetch returned undefined) — silent warning in output channel, no error toast

The `catch` block handles actual failures (network errors, checksum mismatches from _downloadAndVerify) — these MUST show a user-facing error per CONTEXT.md.

Do NOT modify the reuse path (lockfile + health check) or the spawn path below — those are unchanged per PRES-01.
