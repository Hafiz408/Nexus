Output: Updated `.github/workflows/build.yml` with a `github-release` job and a leaner `package` job; updated `.vscodeignore` with `bin/**` exclusion.

<task id="37-01-T1" title="Add github-release job to build.yml" type="auto">
  <name>Task 1: Add github-release job to build.yml</name>
  <files>.github/workflows/build.yml</files>
  <read_first>
    - .github/workflows/build.yml — understand full pipeline structure (jobs, needs chains, trigger conditions)
    - .planning/phases/37-release-pipeline/37-CONTEXT.md — archive names, checksum format, constraints
  </read_first>
  <action>
    Edit `.github/workflows/build.yml` to insert a new `github-release` job between the build jobs and the package job.

    **1. Add `permissions` at the workflow level** (after `on:` block, before `jobs:`):
    ```yaml
    permissions:
      contents: write
    ```

    **2. Add the `github-release` job** after `build-win` and before `package`:
    ```yaml
      github-release:
        name: Upload release assets
        runs-on: ubuntu-latest
        needs: [build-mac, build-win]
        if: startsWith(github.ref, 'refs/tags/v')
        steps:
          - uses: actions/checkout@v4

          - name: Download Mac binary
            uses: actions/download-artifact@v4
            with:
              name: nexus-backend-mac
              path: assets/

          - name: Download Windows binary
            uses: actions/download-artifact@v4
            with:
              name: nexus-backend-win
              path: assets/

          - name: Generate checksums
            run: |
              cd assets
              sha256sum nexus-backend-mac.tar.gz nexus-backend-win.tar.gz > checksums.sha256
              echo "Generated checksums:"
              cat checksums.sha256

          - name: Create GitHub Release and upload assets
            env:
              GH_TOKEN: ${{ github.token }}
            run: |
              TAG="${GITHUB_REF#refs/tags/}"
              gh release create "$TAG" \
                --title "$TAG" \
                --generate-notes \
                assets/nexus-backend-mac.tar.gz \
                assets/nexus-backend-win.tar.gz \
                assets/checksums.sha256
    ```

    Key design points:
    - `if: startsWith(github.ref, 'refs/tags/v')` ensures release upload only on tag push, not workflow_dispatch
    - `gh release create` uploads all three files atomically in one command
    - `sha256sum` produces the exact `<hash>  <filename>` format Phase 36 expects
    - `GH_TOKEN: ${{ github.token }}` uses the built-in GITHUB_TOKEN (no extra secret needed) with `contents: write` permission
    - Assets download into `assets/` directory (not `extension/bin/`) to avoid confusion with the extension's bin path
  </action>

  <action>
    **1. Update `extension/.vscodeignore`** — add `bin/**` line to exclude all native binaries from the VSIX package:

    Append the following line to the end of the file:
    ```
    bin/**
    ```

    **2. Update the `package` job in `.github/workflows/build.yml`:**

    a. Change `needs:` from `[build-mac, build-win]` to handle both tag and workflow_dispatch flows:
       ```yaml
       needs: [build-mac, build-win, github-release]
       if: always() && !failure() && !cancelled()
       ```
       On tag push: all three jobs complete, then package runs. On workflow_dispatch: github-release is skipped (its `if` is false), but package still runs because `always()` ignores skipped dependencies while `!failure() && !cancelled()` still blocks on real failures.

    b. **Remove** the two `Download Mac binary` and `Download Windows binary` steps (the `actions/download-artifact@v4` steps that download into `extension/bin/`). The VSIX must NOT contain binaries.

    The final `package` job should look like:
    ```yaml
      package:
        name: Package .vsix
        runs-on: ubuntu-latest
        needs: [build-mac, build-win, github-release]
        if: always() && !failure() && !cancelled()
        steps:
          - uses: actions/checkout@v4

          - name: Set up Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'

          - name: Install extension dependencies
            run: cd extension && npm ci

          - name: Build extension
            run: cd extension && npm run build

          - name: Package .vsix
            run: |
              cd extension
              npm install -g @vscode/vsce
              vsce package --out nexus.vsix

          - name: Upload .vsix
            uses: actions/upload-artifact@v4
            with:
              name: nexus-vsix
              path: extension/nexus.vsix
    ```
  </action>
