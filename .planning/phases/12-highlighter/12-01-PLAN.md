  <action>
Create `extension/src/HighlightService.ts` implementing the `HighlightService` class. The class must:

1. In the constructor, create exactly ONE `TextEditorDecorationType` (HIGH-02 — never create per query, causes memory leak):
   ```typescript
   this._decorationType = vscode.window.createTextEditorDecorationType({
     backgroundColor: new vscode.ThemeColor('editor.findMatchHighlightBackground'),
     isWholeLine: true,
   });
   ```

2. Implement `async highlightCitations(citations: Citation[]): Promise<void>`:
   - Call `this.clearHighlights()` first — cancels any pending timer AND clears old decorations (HIGH-02: new query replaces old highlights)
   - Group citations by `file_path` using a `Map<string, Citation[]>`
   - For each file group:
     - `const doc = await vscode.workspace.openTextDocument(vscode.Uri.file(filePath))`
     - `const editor = await vscode.window.showTextDocument(doc, { preserveFocus: true, preview: false })` — preserveFocus avoids stealing user focus; preview: false avoids disposable preview tab
     - Map citations to `vscode.Range` objects: `new vscode.Range(new vscode.Position(Math.max(0, c.line_start - 1), 0), new vscode.Position(Math.max(0, c.line_end - 1), Number.MAX_SAFE_INTEGER))` — line_start/line_end are 1-indexed; VS Code Position is 0-indexed (subtract 1)
     - Wrap the entire per-file block in try/catch — file may not exist on disk, skip silently
     - Call `editor.setDecorations(this._decorationType, ranges)` inside the try block
   - After looping all files, schedule: `this._clearTimer = setTimeout(() => this.clearHighlights(), 10_000)` (HIGH-02: auto-clear after 10 seconds)

3. Implement `clearHighlights(): void`:
   - If `this._clearTimer !== undefined`, call `clearTimeout(this._clearTimer)` and set `this._clearTimer = undefined`
   - For each editor in `vscode.window.visibleTextEditors`, guard with `if (editor.document)` before calling `editor.setDecorations(this._decorationType, [])` (guard prevents "setDecorations on invisible editor" warning — race condition documented in microsoft/vscode#18797)

4. Implement `dispose(): void`:
   - Call `this.clearHighlights()`
   - Call `this._decorationType.dispose()` — frees extension host resources on deactivation

Private fields:
- `private readonly _decorationType: vscode.TextEditorDecorationType`
- `private _clearTimer: ReturnType<typeof setTimeout> | undefined`

Import: `import * as vscode from 'vscode';` and `import { Citation } from './types';`
