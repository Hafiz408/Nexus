<objective>
Make parse_file() thread-safe by moving Parser construction inside the function body, so concurrent calls from asyncio.to_thread do not share mutable Parser state across threads.

Purpose: The research identified that ast_parser.py uses module-level Parser singletons (py_parser, ts_parser, tsx_parser). Tree-sitter Parser objects have internal mutable state and are not safe for concurrent use from multiple threads. When pipeline.py calls parse_file() via asyncio.to_thread with 10 concurrent workers, all threads share these Parser instances, risking segfaults or corrupted ASTs.
Output: Updated backend/app/ingestion/ast_parser.py with per-call Parser construction.
</objective>
