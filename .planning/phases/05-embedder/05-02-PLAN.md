<task type="auto">
  <name>Task 2: Implement embedder.py with pgvector + FTS5 init and embed_and_store</name>
  <files>backend/app/ingestion/embedder.py</files>
  <action>
Create `backend/app/ingestion/embedder.py` with these public functions:

**`init_pgvector_table() -> None`** (EMBED-02):
- Call `get_db_connection()` from `app.db.database` — this already calls `conn.autocommit = True`
- Call `register_vector(conn)` (per-connection, not global)
- CREATE TABLE IF NOT EXISTS `code_embeddings`:
  ```sql
  CREATE TABLE IF NOT EXISTS code_embeddings (
      id         TEXT PRIMARY KEY,
      repo_path  TEXT NOT NULL,
      name       TEXT NOT NULL,
      file_path  TEXT NOT NULL,
      line_start INT,
      line_end   INT,
      embedding  vector(1536)
  )
  ```
- CREATE INDEX IF NOT EXISTS `idx_code_embeddings_ivfflat` ON code_embeddings USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100)
- Close conn. Since `conn.autocommit = True` (set by `get_db_connection()`), no explicit commit needed.

**`_sqlite_db_path() -> str`** — internal helper, returns `"data/nexus.db"`. Same file as graph_store.py.

**`_init_fts_table(db_path: str) -> None`** — internal (EMBED-03):
- `conn = sqlite3.connect(db_path)`, CREATE VIRTUAL TABLE IF NOT EXISTS `code_fts` USING fts5:
  ```sql
  CREATE VIRTUAL TABLE IF NOT EXISTS code_fts
  USING fts5(node_id UNINDEXED, name, file_path UNINDEXED, content='')
  ```
- `conn.commit()`, `conn.close()`

**`embed_and_store(nodes: list[CodeNode], repo_path: str) -> int`** (EMBED-01, EMBED-04, EMBED-05, EMBED-06):
- Initialize OpenAI client lazily INSIDE this function body (not at module level) to avoid ValidationError during test imports when OPENAI_API_KEY is absent: `client = OpenAI(api_key=get_settings().openai_api_key)`
- Get pg conn, call `register_vector(conn)`, get db_path, call `_init_fts_table(db_path)`, open sqlite conn
- `total_stored = 0`
- Loop in batches of 100 (EMBED_BATCH_SIZE = 100): `for i in range(0, len(nodes), 100)`
  - `batch = nodes[i:i+100]`
  - `texts = [n.embedding_text for n in batch]`
  - Call `client.embeddings.create(model="text-embedding-3-small", input=texts)` (EMBED-04)
  - Sort response by index: `embeddings = [item.embedding for item in sorted(response.data, key=lambda x: x.index)]`
  - Build rows for pgvector upsert: `(n.node_id, repo_path, n.name, n.file_path, n.line_start, n.line_end, emb)` for each n, emb pair
  - Call `execute_values` with ON CONFLICT (id) DO UPDATE (EMBED-05):
    ```sql
    INSERT INTO code_embeddings (id, repo_path, name, file_path, line_start, line_end, embedding)
    VALUES %s
    ON CONFLICT (id) DO UPDATE SET
        repo_path=EXCLUDED.repo_path, name=EXCLUDED.name,
        file_path=EXCLUDED.file_path, line_start=EXCLUDED.line_start,
        line_end=EXCLUDED.line_end, embedding=EXCLUDED.embedding
    ```
  - FTS5 upsert (EMBED-03): DELETE then INSERT for this batch (FTS5 has no ON CONFLICT):
    - `sqlite_conn.executemany("DELETE FROM code_fts WHERE node_id = ?", [(n.node_id,) for n in batch])`
    - `sqlite_conn.executemany("INSERT INTO code_fts(node_id, name, file_path) VALUES (?,?,?)", [(n.node_id, n.name, n.file_path) for n in batch])`
    - `sqlite_conn.commit()`
  - `total_stored += len(batch)`
- Close pg conn, close sqlite conn
- Return `total_stored` (EMBED-06)

**Module-level constant:** `EMBED_BATCH_SIZE = 100`
