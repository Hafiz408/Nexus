Create `backend/app/ingestion/graph_store.py` implementing the three STORE requirements.

**SQLite path strategy:** Use a single `data/nexus.db` file with a `repo_path` TEXT column on all tables (single-file approach recommended in RESEARCH.md — simpler than per-repo files).

**Schema (graph_nodes table):**
```sql
CREATE TABLE IF NOT EXISTS graph_nodes (
    node_id    TEXT NOT NULL,
    repo_path  TEXT NOT NULL,
    file_path  TEXT NOT NULL,     -- separate column for efficient delete_nodes_for_files
    attrs_json TEXT NOT NULL,     -- full node attribute dict from node.model_dump() + pagerank/degrees
    PRIMARY KEY (node_id, repo_path)
)
```

**Schema (graph_edges table):**
```sql
CREATE TABLE IF NOT EXISTS graph_edges (
    source    TEXT NOT NULL,
    target    TEXT NOT NULL,
    repo_path TEXT NOT NULL,
    attrs_json TEXT NOT NULL DEFAULT '{}',
    PRIMARY KEY (source, target, repo_path)
)
```

**Implementations:**

`_get_conn(db_path: str) -> sqlite3.Connection` — internal helper: connects, creates both tables with IF NOT EXISTS, returns conn. Call `conn.commit()` after CREATE TABLE statements.

`_db_path() -> str` — returns `"data/nexus.db"` (hardcoded; V1 uses single file).

`save_graph(G: nx.DiGraph, repo_path: str) -> None` (STORE-01):
- Get conn, clear existing rows for this repo_path: `DELETE FROM graph_nodes WHERE repo_path = ?` and `DELETE FROM graph_edges WHERE repo_path = ?`
- `executemany` INSERT for nodes: `(node_id, repo_path, attrs["file_path"], json.dumps(attrs))`. Attrs come from `G.nodes(data=True)` — these are already plain dicts (graph_builder stores model_dump() results + pagerank floats). If any value is not JSON-serializable, convert with a default: `json.dumps(attrs, default=str)`.
- `executemany` INSERT for edges: `(u, v, repo_path, json.dumps(edge_attrs, default=str))`
- `conn.commit()`, `conn.close()`

`load_graph(repo_path: str) -> nx.DiGraph` (STORE-02):
- Get conn, SELECT all from graph_nodes WHERE repo_path = ?, `G.add_node(row["node_id"], **json.loads(row["attrs_json"]))` for each
- SELECT all from graph_edges WHERE repo_path = ?, `G.add_edge(row["source"], row["target"], **json.loads(row["attrs_json"]))` for each
- `conn.close()`, return G
- Use `conn.row_factory = sqlite3.Row` for named column access

`delete_nodes_for_files(file_paths: list[str], repo_path: str) -> None` (STORE-03):
- Get conn
- For each affected node_id: SELECT node_id FROM graph_nodes WHERE repo_path = ? AND file_path IN (placeholders)
- DELETE FROM graph_edges WHERE repo_path = ? AND (source IN (node_ids) OR target IN (node_ids))
- DELETE FROM graph_nodes WHERE repo_path = ? AND file_path IN (placeholders)
- `conn.commit()`, `conn.close()`
- Handle empty `file_paths` list early return (no-op)

**Key pitfalls to avoid (from RESEARCH.md):**
- node attributes in graph_builder store plain dict values (model_dump() was called in Pass 1) so json.dumps is safe, but use `default=str` as a safety net for pagerank floats
- `file_path` column enables O(n) deletion without JSON parsing — do not use LIKE on attrs_json
- Use `sqlite3.Row` factory for readable column access
